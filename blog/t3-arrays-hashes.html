Arrays & Hashes in Ruby



12/5/14


This week we began our journey into Ruby, the programming language of all programming languages. It has been a great experience. Ruby is prized for being relatively intuitive as far as programming languages go. I want to highlight two key features that make storing and accessing data incredibly simple, Arrays and Hashes.


Let’s start with Arrays. Arrays are incredibly useful because they can hold a list of objects. Say your friend Johnny is an incredibly picky eater. You can store all of the foods Johnny will eat in an array called “johnny_food” like so:


johnny_food = [“pizza”, “spaghetti”, “burritos”, “cheeseburgers”, “tacos”, “french fries”, “lobster”]

Now just call the array


johnny_food
=> [“pizza”, “spaghetti”, “burritos”, “cheeseburgers”, “tacos”, “french fries”, “lobster”]

Why don’t you try some kale Johnny, huh?


So if we can’t remember what Johnny’s foods are, lets just call the array “johnny_food” and …BAM!...there’s our list.


Ok now let’s assume Johnny is on a diet. He has to eat one of the foods on his list on a certain day of the week (…some diet Johnny!). Lets say that our array is in order, starting on Monday, so that Monday he eats pizza, Tuesday he eats spaghetti, etc. So how do we access these values? It’s super easy. Array’s are indexed by numbers, just keep in mind that the starting number is 0. So the seven days of the week could be represented by index numbers 0-6.


For example: What does Johnny eat on Thursday? Thursday = day 4 = index number[3].


johnny_food[3] = cheeseburgers

Easy as that. Now lets take a look at hashes. Hashes are similar to array in the sense that they store multiple objects, the main difference is that a hash needs a KEY and a VALUE.


Here’s an example. Johnny has a bunch of really picky friends, who are on the same diet as him. Your having this horrible group of people over for dinner, how do you keep it straight? Let’s make a hash.


The format looks like this:


hash = {
key => value
key => value
}

so…


picky_eater = {

“johnny” => “pizza”
“timmy” => “spaghetti”
“jimmy” => “burritos”
“billy” => “cheeseburgers”
}

What was it that Jimmy needed to eat today? puts picky_eater[“jimmy”] and it will return “burritos”. Pretty cool right? It keeps your information nicely organized.


Disclaimer: No one actually had to cook for these horribly picky eaters, and no one ever should.
